## Executive summary

| Verdict | Detail |
|---|---|
| **Architecture holds** | Hub-and-spoke (types / industries / use cases) is sound. Money is in **type tools** + a few **use-case** pages, not industry exact-match phrases. |
| **Head term is a long game** | `qr code generator` = **1,000,000** MSV, **KD 96**. Home page should target it, but a new domain will not win it soon. |
| **Product differentiators are real demand** | `free qr code generator no sign up` **4,400**; `no expiration` **2,900**; `with logo` **2,400**. Lean hard on these in titles/H1s. |
| **Brief demand tiers need recalibration** | Several “High” industry keywords are tiny exact MSV; several “Med/Low” type keywords (Instagram, Spotify, Bitcoin, YouTube) are stronger than labeled. |
| **Best early wins (low–mid KD + real volume)** | WiFi, vCard, Google Review, Instagram, business card, menu (as *qr code menu*), WhatsApp, Facebook, YouTube. |
| **Do not build for “dynamic” yet** | `dynamic qr code generator` = **2,400** MSV — correctly excluded until product supports it. |
| **Locale seeds mostly correct** | Biggest surprises: **PT-BR, DE, ID, JA, PL, IT** head-term shape (see §4). |

**Priority score used below:** rough `MSV × (100 − KD) / 100` on the best primary for that URL (US). Higher = better near-term opportunity for a new domain.

## 11. Bottom line

The brief’s **taxonomy and product-led angles are good**. Validation changes **priority and primaries**, not the whole IA:

1. **Double down** on WiFi, vCard, business card, Google Review, Instagram, menu, social type tools.  
2. **Retarget** menu + restaurant + small-business primaries.  
3. **Demote** weak industry exact-matches and low-MSV feature pages.  
4. **Upgrade** Spotify, YouTube, Facebook, Bitcoin vs brief tiers.  
5. **Localize in order** DE → ID → PT-BR → JA → PL → IT before smaller ES/FR/UK.  
6. Keep **no sign-up / no expiry / logo / free** as the conversion layer on every money page — those modifiers have verified demand and match the product.
